# v0.3.3

## Bug Fixes

### Logcat / HiLog ‚Äî Scrollbar Drag Now Pauses Auto-Scroll

**Problem**: dragging the scrollbar thumb upward did not pause auto-scroll. The next DOM flush snapped the view back to the bottom, making it impossible to read earlier log entries using the scrollbar.

**Root cause**: `handleScroll` only re-enabled auto-scroll (when reaching the bottom) ‚Äî it never disabled it. The only path that disabled auto-scroll was `handleWheel` (`deltaY < 0`), which does not fire on scrollbar drag.

**Fix**: `handleScroll` now also disables auto-scroll when the scroll position is not at the bottom (guards on `autoScrollRef.current` to fire exactly once per pause transition). This covers mouse wheel, scrollbar drag, keyboard scrolling, and any other scroll mechanism.

### Shell Terminal ‚Äî Auto-Scroll Pause

**Problem**: the Shell terminal had no auto-scroll pause mechanism at all. Every incoming data chunk triggered a `flushToDOM` call that unconditionally set `scrollTop = scrollHeight`, overriding any scroll position the user had set. Logs were continuously snapping to the bottom, making them unreadable during active output.

**Fix**: added a full auto-scroll pause mechanism to the Shell terminal, matching the behavior of Logcat and HiLog:
- Mouse wheel up or scrollbar drag pauses auto-scroll; DOM updates are fully suspended (data continues accumulating in `htmlStringMap` up to the configured line limit).
- A **‚Üì Bottom** button appears in the toolbar when paused; clicking it (or scrolling back to the bottom) flushes the latest buffered output and resumes auto-scroll.
- Switching devices resets auto-scroll to on.

### Shell Terminal ‚Äî Scrollbar Width

The Shell terminal output area now uses an 8 px scrollbar (vs. 6 px global default), making it easier to grab during active log output.
