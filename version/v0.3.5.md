# v0.3.5

## New Features

### Quick Commands — Command Groups (Directory View)

**Problem**: When users accumulate many quick commands, the flat list becomes hard to navigate. There was no way to organise commands by purpose or workflow.

**Solution**: Added a VS Code-style collapsible directory view to the Quick Commands panel. Commands can be organised into custom **groups** (folders) with a single level of nesting.

**Group management**:
- **New Group** button at the bottom of the command list creates a group and pre-selects it in the add form
- Group headers show the command count and support collapse/expand via the chevron icon
- **Double-click** a group label to rename it inline; press Enter or click away to confirm
- Hover over a group header to reveal a `+` button (add command to this group) and a `⋯` menu (Rename / Delete Group)
- Deleting a group moves all its commands to ungrouped — no commands are lost

**Command assignment**:
- The add-command / add-script form has a **Group** selector (optional; defaults to ungrouped)
- Each command row has a `⋯` dropdown with a **Move to** submenu to reassign the command to any group or back to ungrouped at any time

**Persistence**: Group structure and collapse state are persisted to localStorage (store key `"bridge-commands"`, version `2`). Existing command data from v1 migrates automatically — all commands appear ungrouped, and groups can be created from scratch.

## Bug Fixes

### OHOS Remote Control — OK Button Unresponsive

**Problem**: The OK button in the REMOTE panel sent `uinput -K -d 2016 -u 2016` (KEYCODE_DPAD_CENTER), which is unresponsive on most OpenHarmony devices.

**Fix**: The OK button now maps to `KEYCODE_ENTER (2054)` instead. Confirmed working on OHOS virtual device; DPAD_CENTER (2016) and NUMPAD_ENTER (2064) did not respond.
