# Version 0.2.0

## New Features

### 1. Drag-and-drop file upload

**Implementation:** Both ADB FileManager and OHOS HdcFileManager now support dragging files from the OS file explorer directly into the file table area. When files are dragged over the table, a translucent overlay with a dashed border appears showing "Drop files to upload to {currentPath}". On drop, files are uploaded to the currently displayed remote directory.

**Technical approach:**
- Listens for Tauri's window-level `tauri://drag-enter`, `tauri://drag-leave`, and `tauri://drag-drop` events via `@tauri-apps/api/event`.
- A `dragCountRef` counter handles nested enter/leave events to correctly track drag state.
- When the Upload modal is open, direct-drop is suppressed — the modal handles its own drag-drop instead.
- The drop event provides native file paths which are passed to `pushFiles`/`sendHdcFiles`.

### 2. Improved Upload dialog with drag-drop and custom path

**Implementation:** The "Upload" button now opens a modal dialog (`UploadModal`) instead of directly opening a file picker. The modal features:
- A **drop zone** area where users can drag files directly from the OS, or click to open a file browser.
- A **destination path** input field, pre-filled with the current directory, allowing users to customize the upload target.
- A **file list** showing all selected files with individual remove buttons.
- **Deduplication**: dragging or browsing the same file twice won't add duplicates.

**Technical approach:**
- Created a shared `src/components/shared/UploadModal.tsx` component used by both ADB and OHOS file managers.
- The modal accepts an `onUpload` callback prop, keeping the upload logic in the parent component.
- Tauri drag-drop events are listened for only while the modal is open, and cleaned up on close.
- UI uses theme-aware CSS variables (`var(--border)`, `var(--accent)`, `var(--text-secondary)`) for consistent dark/light theme integration.

### 3. Multi-selection with batch operations

**Implementation:** File Manager now supports multi-selection of files and folders:

**Selection behavior (Windows-like):**
- **Single-click**: Toggles selection on the clicked item (select if unselected, deselect if selected).
- **Double-click**: Clears all selections first, then opens the item — folders navigate into the directory, files open the View modal.
- Uses a 250ms click timer to differentiate single-click (toggle) from double-click (open).

**Batch operations:**
- **Download**: When multiple items are selected, opens a directory picker and downloads all selected items to that directory. Single file selection still uses a save-as dialog.
- **Delete**: Shows a confirmation dialog with the count of items to delete, then deletes all selected items sequentially.
- Toolbar buttons show selection count: "Download (3)", "Delete (3)".

**View button removed:** Since double-clicking a file now triggers the View modal, the dedicated "View" button has been removed from the toolbar.
