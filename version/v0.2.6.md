# v0.2.6

## Fixes

### Transfer queue progress bar not matching dark theme

**Problem:** The file upload progress bar container in `TransferQueue` had hardcoded `background: "#fff"` and `borderTop: "1px solid #f0f0f0"`, always rendering as white regardless of the active theme. The progress bar trail (unfilled portion) also used Ant Design's default light color.

**Fix:** Replaced hardcoded colors with CSS variables: `var(--content-bg)` for background, `var(--border)` for the top border, and added `trailColor="var(--card-bg)"` to the `Progress` component so it adapts to all themes.

### OHOS device disconnect very slow

**Problem:** `disconnect_device()` called `hdc tconn <addr> -remove` without first stopping any running hilog, tlogcat, or shell stream processes. These child processes held open connections to the device, causing the HDC disconnect command to hang until they timed out.

**Fix:** Added `kill_streams_for_device()` (hilog.rs) and `kill_shell_stream()` (commands.rs) helper functions. `disconnect_device()` now calls both via `tokio::join!` before executing the disconnect command, so all child processes are terminated first.

### Reconnected OHOS device not re-remounted after reboot

**Problem:** The device watcher's `attempted_remounts` HashSet tracked which devices had been remounted, but never cleared entries when a device went offline (e.g., after reboot or unexpected disconnect). When the device reconnected with the same connect_key, the watcher skipped auto-remount because the key was already in the set. The `DEVICE_REMOUNT_STATUS` cache also retained stale data.

**Fix:** The device watcher now builds a set of currently connected device keys each poll cycle and clears `attempted_remounts` entries (and their `DEVICE_REMOUNT_STATUS` cache) for any device that is no longer connected. When the device reconnects, its key is absent from both sets, triggering a fresh remount attempt.

### No visual feedback during OHOS device disconnect

**Problem:** When disconnecting an OHOS device (which can take several seconds), the UI showed no feedback, making it appear stuck.

**Fix:** Added a `disconnecting` state set in the Sidebar. While a disconnect is in progress, the disconnect icon is replaced with a spinning `LoadingOutlined` indicator. The spinner is removed when the operation completes (success or failure).
