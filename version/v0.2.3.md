# v0.2.3 ‚Äî Quick Commands Improvements

## Features & Fixes

### 1. Quick Commands persistent storage

**Problem:** Quick Commands were stored in a plain Zustand store without persistence. All custom commands were lost on app restart.

**Fix:** Added `persist` middleware to `commandStore.ts` using `zustand/middleware`, storing to `bridge-commands` via `tauri-plugin-store`. Commands now survive app restarts.

### 2. Per-device-type Quick Commands

**Problem:** All device types (ADB, OHOS, Serial) shared the same Quick Commands list. ADB-specific commands like `pm list packages` are not applicable to OHOS or Serial devices.

**Implementation:** Restructured the store from a flat `commands: QuickCommand[]` array to `commandsByType: { adb, ohos, serial }` ‚Äî each device type has its own independent command list. The `QuickCommandsPanel` automatically shows the correct list based on the currently selected device's type, indicated by a colored tag (blue for ADB, green for OHOS, orange for Serial). Each type has its own default commands (ADB: Reboot/Get Props/List Packages; OHOS: Reboot/Device Info; Serial: empty).

Also fixed a bug where OHOS devices were treated as serial devices in the Quick Commands panel ‚Äî they now correctly use `startHdcShellStream` instead of falling through to the serial `writeToPort` path.

### 3. Local script (.bat) execution from Quick Commands

**Problem:** The user wanted to execute local batch scripts (.bat files) from the Quick Commands panel for various devices.

**Implementation:**
- Added a `scriptPath` optional field to `QuickCommand` ‚Äî when set, the command executes a local script instead of sending a shell command to the device.
- Added "Add Script" button in the Quick Commands panel that opens a file picker (supports .bat, .cmd, .ps1, .sh files).
- Backend: New `run_script` / `stop_script` functions in `util.rs` that spawn `cmd /C <script>` with streamed stdout/stderr via `script_output` / `script_exit` Tauri events.
- Script output appears in the Shell terminal alongside device output.
- Scripts are tagged with a purple "script" badge in the command list.
