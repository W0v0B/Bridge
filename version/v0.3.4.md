# v0.3.4

## New Features

### ADB Screen Mirror — Screenshot Fallback Mode

**Problem**: Not all Android devices support Scrcpy (e.g., devices without `FEATURE_SECURE_HARDWARE_ATTESTATION`, some older ROMs, or restricted environments). Previously, users on unsupported devices had no screen mirror option.

**Solution**: Added a **Screenshot** mode to the ADB Screen Mirror panel alongside the existing **Scrcpy** mode. Users can toggle between modes via a segmented control in the toolbar; the selected mode persists to `localStorage`.

Screenshot mode uses `adb exec-out screencap -p` to capture a PNG frame directly from stdout (no temp file written to the device). A configurable capture interval (0.2–3 fps), FPS counter, and in-app image display are provided — matching the UX of the existing OHOS Screen Mirror panel.

**Backend** (`src-tauri/src/adb/screen.rs`): new module using the same session-map + `Arc<AtomicBool>` cancellation pattern as the OHOS mirror. Emits `adb_screen_frame` (base64 PNG) and `adb_screen_state` events. Session is also cleaned up on device disconnect.

**New Tauri commands**: `start_adb_screen_capture`, `stop_adb_screen_capture`, `is_adb_screen_capture_running`.

## Bug Fixes

### OHOS Remote Control — Buttons Had No Effect

**Problem**: The REMOTE panel buttons (D-pad, Home, Back, Menu, Volume, Power) did nothing on OHOS devices. The panel was sending `input keyevent <android_keycode>`, which is an Android-only command — OpenHarmony does not have an `input` binary.

**Fix**: OHOS devices use `uinput -K -d <code> -u <code>` for input injection. Additionally, OHOS uses a completely different keycode numbering system (`@ohos.multimodalInput.keyCode`): HOME=1, BACK=2, DPAD=2012–2016, VOL_UP/DOWN=16/17, POWER=18, MENU=2067.

The `sendKey` callback in `HdcScreenMirrorPanel.tsx` now translates Android keycodes to OHOS keycodes via `OHOS_KEYCODE_MAP` (in `src/utils/hdc.ts`) and sends the correct `uinput` command.
