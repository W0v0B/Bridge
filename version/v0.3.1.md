# v0.3.1

## Features

### OHOS Screen Mirror

Mirror OHOS (HarmonyOS / OpenHarmony) device screens directly inside DevBridge. Unlike the ADB screen mirror (which launches an external scrcpy window), OHOS screen mirror renders frames in-app as a live JPEG image.

**Backend** (`src-tauri/src/hdc/screen.rs` — new file):
- Capture loop (tokio task): runs `hdc -t {connectKey} shell snapshot_display -f <remote>` → `hdc -t {connectKey} file recv <remote> <local_temp>` → base64-encode → emit `hdc_screen_frame { connect_key, data }`
- Configurable interval: 333–5000 ms (clamped), persisted to `localStorage` key `"hdc_screen_config"`
- Cancellation via `Arc<AtomicBool>` flag stored in `SCREEN_SESSIONS` map; loop exits cleanly on next iteration
- Auto-exits after 5 consecutive snapshot or transfer failures
- On exit: deletes local temp file and remote `/data/local/tmp/devbridge_screen.jpeg`, emits `hdc_screen_state { running: false }`
- `kill_session(connect_key)` — best-effort stop called automatically by the device watcher on disconnect

**New Tauri commands** (registered in `lib.rs`):
- `start_hdc_screen_mirror(connectKey, config)` — starts capture loop; stops any existing session first
- `stop_hdc_screen_mirror(connectKey)` — sets cancellation flag; no-op if not running
- `is_hdc_screen_mirror_running(connectKey)` — synchronous map lookup

**New events**:
- `hdc_screen_frame` → `ScreenFrame { connect_key, data }` — one per captured frame
- `hdc_screen_state` → `HdcScreenMirrorState { connect_key, running }` — on start and stop

**Frontend** (`src/components/hdc/HdcScreenMirrorPanel.tsx` — new file):
- Start/Stop button with loading state synced via `hdc_screen_state` events
- FPS counter (frames counted per second, reset on stop)
- Interval slider (3fps / 2fps / 1fps / 0.5fps / 0.2fps marks) — disabled while running
- In-app image display area showing the latest frame; placeholder icon when stopped
- Remote control panel (see below)

**New "Screen Mirror" tab** added to the OHOS tab bar (between HiLog and File Manager).

### Remote Control Panel (shared component)

A D-pad + utility key remote for sending `input keyevent` commands to connected devices. Shared between both the OHOS and ADB screen mirror panels.

**Shared component** (`src/components/shared/RemoteControlPanel.tsx` — new file):
- D-pad: Up / Down / Left / Right / OK (3×3 CSS grid, corners empty)
- Utility row 1: Home / Back / Menu
- Utility row 2: Vol+ / Vol− / Power
- Props: `disabled: boolean`, `onSendKey: (keyCode: number) => Promise<void>`
- Android KeyEvent codes (compatible with OHOS): HOME=3, BACK=4, DPAD_UP=19, DPAD_DOWN=20, DPAD_LEFT=21, DPAD_RIGHT=22, DPAD_CENTER=23, VOLUME_UP=24, VOLUME_DOWN=25, POWER=26, MENU=82

**ADB Screen Mirror** (`src/components/adb/ScreenMirrorPanel.tsx` — updated):
- Remote control panel added alongside the scrcpy settings panel (side-by-side layout)
- Each button sends `input keyevent <code>` via the existing `runShellCommand(serial, command)`
- Remote disabled when no ADB device is selected

**OHOS Screen Mirror** (`src/components/hdc/HdcScreenMirrorPanel.tsx`):
- Inline remote panel extracted and replaced with `<RemoteControlPanel>`
- Each button sends `input keyevent <code>` via `runHdcShellCommand(connectKey, command)`
- Remote disabled when no OHOS device is selected
