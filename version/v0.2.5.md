# v0.2.5

## Fixes

### Config store migration for new fields

**Problem:** Users with persisted config from before v0.2.4 (when `adbQuickPaths` / `ohosQuickPaths` were added) got `undefined` for these fields because zustand's default shallow merge replaced the entire `config` object with the persisted one, losing newly added defaults. This caused a crash (`Cannot read properties of undefined (reading 'map')`) when the File Manager tried to render the quick access tags.

**Fix:** Added a custom `merge` function to the zustand `persist` config that deep-merges `config` fields, so new defaults are preserved when the persisted state doesn't include them.

### File Manager sort toggle stuck

**Problem:** Ant Design Table's default sort cycle includes a third "cancel" state (`ascend -> descend -> undefined`). When the user clicked a column header three times, `s.order` became `undefined` and the fallback `?? "ascend"` reset the internal state to ascending, but the column header showed no sort indicator — creating a desync.

**Fix:** Added `sortDirections: ["ascend", "descend", "ascend"]` to the Name and Modified columns in both ADB FileManager and OHOS HdcFileManager, so clicking toggles directly between ascending and descending without the cancel state.

### Quick access path deletion removing multiple tags

**Problem:** Two issues: (1) `key={idx}` caused incorrect React reconciliation when removing items from the middle of the array — React would unmount the wrong Tag. (2) Missing `e.preventDefault()` on the Ant Design Tag's `onClose` allowed its built-in close animation to remove the tag from the DOM independently of React state, causing multiple tags to disappear.

**Fix:** Changed `key` to `${qp.label}:${qp.path}` for stable identity, and added `e.preventDefault()` so only the React state update controls removal. Applied to both ADB FileManager and OHOS HdcFileManager.

### Duplicate script event listeners causing double output

**Problem:** Both `ShellPanel` and `QuickCommandsPanel` listened for `script_output` and `script_exit` events, causing script output to appear twice in the terminal when Quick Commands was visible.

**Fix:** Removed the duplicate listeners from `QuickCommandsPanel` — `ShellPanel` handles all script events.
